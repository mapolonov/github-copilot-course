## 1. Що таке GitHub Copilot — чесний технічний погляд

### 1.1 Архітектура на високому рівні

GitHub Copilot — це не "пошуковик по коду". Це **inference endpoint** над LLM (на базі сімейства моделей від OpenAI, Anthropic, Google та інших провайдерів), навченої на:
- Публічних репозиторіях GitHub (до моменту зрізу навчання)
- Ліцензійних датасетах коду
- Кількох додаткових джерелах (документація, Stack Overflow тощо)

```
Ваш редактор
    │
    ├── Поточний файл (до/після курсора) ──┐
    ├── Відкриті вкладки (сусідні файли)    ├──► Context Window ──► LLM ──► Suggestion
    ├── Коментарі / Chat історія            │         (~8-32K токенів)
    └── .github/copilot-instructions.md ──┘
```

**Ключове розуміння:** Copilot не "думає" і не "розуміє" ваш бізнес. Він передбачає найбільш імовірне продовження тексту на основі патернів з навчальних даних.

### 1.2 Типи взаємодії

| Режим | Як працює | Коли використовувати |
|-------|-------------|-------------------|
| **Inline Completion** | Автодоповнення в редакторі | Написання коду "в потоці" |
| **Copilot Chat** | Діалог в окремій панелі | Складні питання, пояснення, рефакторинг |
| **Inline Chat** | `Ctrl+I` прямо у файлі | Швидкі локальні зміни |
| **Copilot Edits** | Мультифайлові зміни | Feature-level задачі |
| **@workspace** | Контекст всього проєкту в чаті | Аналіз архітектури, пошук по проєкту |

## 2. Як Copilot розуміє ваш .NET код

### 2.1 Контекстне вікно — найважливіша концепція

```
┌─────────────────────────────────────────────────────────┐
│                    КОНТЕКСТНЕ ВІКНО                      │
│                                                          │
│  [PREFIX]                    [SUFFIX]                    │
│  Все що ДО курсора           Все що ПІСЛЯ курсора        │
│                                                          │
│  - using statements          - Закриваючі дужки          │
│  - Namespace                 - Інші методи класу         │
│  - Class declaration         - Сусідні файли (вкладки)   │
│  - Попередні методи          - Інтерфейси/контракти      │
│  ↓                           ↓                          │
│  Cursor Position ────────────────────────────────────►  │
│                                                          │
│  ВАЖЛИВО: Чим більше релевантного контексту у вікні,     │
│           тим точніша пропозиція!                        │
└─────────────────────────────────────────────────────────┘
```

### 2.2 Що Copilot бачить у вашому .NET проєкті

**Добре бачить:**
- Імена класів, методів, змінних (naming is everything!)
- XML-коментарі над методами
- Відкриті в редакторі файли (інтерфейси, контракти, DTOs)
- `using` директиви — підказують використовувані бібліотеки

**Погано бачить:**
- Закриті файли, які не відкриті в редакторах
- Вміст `appsettings.json` (чутливі дані)
- Бізнес-логіку в БД (збережені процедури, не завантажені в контекст)
- Коментарі в системі задач (Jira, Azure DevOps)

## 3. Чесна розмова про обмеження

### 3.1 Галюцинації — неминуча реальність

Copilot **завжди** буде пропонувати неіснуючі API. Приклади реальних галюцинацій у .NET:

```csharp
// Copilot запропонував цей код (НЕ існує в EF Core 8):
await context.Users
    .SmartFilter(u => u.IsActive)  // ❌ неіснуючий метод
    .ToListAsync();

// Правильний варіант:
await context.Users
    .Where(u => u.IsActive)  // ✅
    .ToListAsync();
```

```csharp
// Copilot запропонував неіснуючий NuGet пакет:
// Install-Package Microsoft.EntityFrameworkCore.BulkExtension
// Правильно: Z.EntityFramework.Plus або EFCore.BulkExtensions
```

**Правило:** Завжди перевіряйте незнайомі API через IntelliSense або документацію.

### 3.2 Security Issues у згенерованому коді

Copilot навчений на публічному коді, в якому багато **антипатернів безпеки**:

```csharp
// ❌ Copilot часто пропонує SQL Injection уразливості:
var query = $"SELECT * FROM Users WHERE Name = '{name}'";

// ✅ Правильно — параметризовані запити:
var users = await context.Users
    .Where(u => u.Name == name)
    .ToListAsync();
```

```csharp
// ❌ Hardcoded credentials — Copilot знає патерни з витоків репозиторіїв:
var connectionString = "Server=prod-db;Password=Admin123;";

// ✅ Завжди через конфігурацію:
var connectionString = configuration.GetConnectionString("DefaultConnection");
```

### 3.3 Training Data Cutoff

Copilot не знає про:
- Нові NuGet пакети, що вийшли після зрізу навчання
- Breaking changes у свіжих версіях .NET
- Внутрішні бібліотеки вашої компанії (без custom instructions)

## 4. Встановлення та налаштування

### 4.1 Visual Studio 2022

```
1. Extensions → Manage Extensions → Search "GitHub Copilot"
2. Встановити "GitHub Copilot" та "GitHub Copilot Chat"
3. Tools → GitHub → Sign in with GitHub Account
4. Перевірка: відкрити будь-який .cs файл, почати друкувати — повинні з'явитися сірі пропозиції
```

**Гарячі клавіші VS 2022:**

| Дія | Hotkey |
|----------|--------|
| Прийняти пропозицію | `Tab` |

| Наступна пропозиція | `Alt+]` |
| Попередня пропозиція | `Alt+[` |
| Відкрити панель пропозицій | `Ctrl+Alt+\` |
| Inline Chat | `Ctrl+I` |
| Відкрити Copilot Chat | `Ctrl+Shift+C` (або іконка в тулбарі) |

## 6. Mentality Shift — как думать о Copilot

### Неправильная ментальная модель ❌
> "Copilot пишет код за меня, я просто проверяю"

### Правильная ментальная модель ✅
> "Copilot — это Senior разработчик с идеальной памятью на паттерны, но без понимания вашего бизнеса. Работает быстро, но требует code review как любой другой разработчик в команде."

### Аналогия с pair programming

```
Традиционный workflow:        Copilot workflow:
───────────────────           ─────────────────
Думаете → Пишете code    →    Думаете → Формулируете намерение
                         →    Copilot предлагает → Вы оцениваете
                         →    Принимаете/корректируете/отклоняете
```

**Скорость растёт от умения формулировать намерения, а не от скорости печати.**

## 7. Метрики — як вимірювати користь

Що відстежує GitHub (в Business/Enterprise):

- **Acceptance Rate** — % прийнятих пропозицій (хороший показник: 25-40%)
- **Lines of Code Accepted** — рядки коду від Copilot
- **Active Users** — хто реально використовує

Що варто вимірювати самостійно:
- Час на написання типового CRUD контролера (до/після)
- Час на написання unit-тестів (до/після — тут Copilot дає max ROI)
- Час на ревю PR (після навчання повинен знизитися, оскількки код став одноріднішим)
